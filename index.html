🧭 Navigation de l'application
📅 Planning
Vue calendrier de tous les postes (Arélia, Événementiel, absences). Filtrez par site, agent, type de poste, ou affichez uniquement les postes déjà validés.
✅ Fiche Validation
Tableau gauche = postes planifiés à cocher. Tableau droit = postes validés (pointage réel). Filtres Tous / Non validés / Changements / Postes extra, plus recherche par agent, site, date et catégorie (Arélia / Événementiel / Autre).
👥 Agents
Création, recherche et gestion des agents (statut APS, SSIAP 1/2/3, Coordinateur, Responsable). Export de la fiche individuelle ou de toutes les fiches en Excel ou PDF, sauvegarde/restauration JSON.
📍 Sites
Liste des sites clients (Arélia ou Événementiel) avec adresse et contact. Recherche et filtre par type, ajout d'un nouveau site.
📋 Commandes
Commandes clients ponctuelles (besoins en agents par site/date/statut). Statut de couverture affiché : ✅ Couverte · 🟠 Partielle · 🔴 Non couverte.
📆 Disponibilités
Déclarations dispo/indispo/partiel des agents, en mode Excel (grille croisée) ou Manuel (saisie agent par agent). Fiches individuelles accessibles via 📋 Fiches Individuel.
⏱️ Heures Supp
4 sous-onglets détaillés ci-dessous : Planning Annuel, HS par Cycle, Planning Arélia, Vérif Comptable.
⚙️ Config
Synchronisation multi-appareils, code de connexion, import de planning, connexions Supabase, gestion des administrateurs, historique des actions.
✅ Détail — Fiche Validation
📥 Import
Depuis ⚙️ Config → Import fichier planning, glissez ou cliquez pour charger votre fichier Excel (.xlsx/.xlsm/.xls), CSV ou PDF. Le planning est extrait automatiquement depuis les onglets Planning ou Arélia, colonnes cachées comprises.
👆 Clic sur ligne droite
Un menu contextuel apparaît sous la ligne avec 3 actions : ✏️ Modifier (heures réelles, statut, motif, scission de poste) · ⇄ Échange (désigner un remplaçant) · 🗑 Supprimer (retirer la validation).
➕ Poste Suppl.
Ajoute un poste non planifié (commande client urgente). Agents et sites proposés en autocomplétion depuis le planning importé.
📊 Export Excel / 📁 Export par catégorie
Export Excel génère un fichier .xlsx avec deux onglets : NOTIFICATIONS (postes validés avec statuts) et RELEVE DYNAMIQUE (planning complet avec coches). Export par catégorie exporte la feuille de relevé en 3 onglets séparés : Arélia / Événementiel / Autre.
☁️ Envoyer / 📥 Charger
Synchronise les validations de pointage avec Supabase (cloud), pour un accès multi-appareils. À configurer dans ⚙️ Config.
🗑 Reset
Efface tout le planning, les validations et les extras du navigateur. Une confirmation est demandée. Irréversible.
Statuts de pointage :
Conforme = présent aux horaires prévus  ·  Retard = arrivée tardive  ·  Échange = remplacé par un autre agent  ·  Absent = non présenté
⏱️ Heures Supp — les 4 sous-onglets
📅 Planning Annuel
Vue annuelle agent × semaine, regroupée par cycles (séparateur orange vertical). Le sélecteur Année couvre désormais une plage glissante d'au moins 10 ans (de l'année en cours − 2 à + 10), pour planifier loin à l'avance. Réglez le Cycle départ, la règle de comptage des postes courts (ex. poste 4h → compté 6h) et sa date d'application, puis cliquez ⚡ Générer.
⏱️ HS par Cycle
Calcule les heures supplémentaires par agent sur des cycles de 4 semaines. 📦 Archiver les plannings des mois passés permet d'importer plusieurs fichiers (Excel, CSV, PDF, photo) en une fois — chaque mois est détecté et archivé automatiquement. 📆 Mois calculés (planning) ajoute des mois calculés depuis le planning importé, en plus de la saisie manuelle. La saisie manuelle permet d'ajouter ou corriger des heures par agent/cycle.
🟢 Planning Arélia
Grille Site × Jour pour le mois et l'année sélectionnés (sélecteurs Mois et Année en haut — l'année couvre également une plage d'au moins 10 ans). 📥 Depuis planning pré-remplit depuis le planning importé. 📋 Copier mois précédent reproduit la même rotation par jour de semaine, avec proposition de générer aussi les mois suivants vides. 🟢 Init. MALQUIN crée les créneaux types du site MALQUIN à compléter.
📊 Vérif Comptable (IDCC 1351)
Compare les calculs KHEOPS à la fiche du comptable. Réglez la date de début de cycle, le jour de départ (1, 9, 16, 26 ou 28 du mois — le comptable calcule souvent à partir du mi-mois précédent), le nombre de semaines (4 ou 5) et le seuil du cycle en heures. La section 🔍 Comparaison avec la fiche comptable permet de saisir ou d'importer (Excel/CSV/PDF/photo) les chiffres du comptable pour détecter automatiquement les écarts par agent.

🔄 Qu'est-ce qu'un cycle ?
Un cycle = 4 semaines consécutives. L'année est découpée en cycles numérotés (C1, C2, C3…). Pour la sécurité privée, les heures supplémentaires ne se calculent pas semaine par semaine mais sur l'ensemble du cycle de 4 semaines.
Année normale = 52 sem. = 13 cycles · Une année à 53 semaines (cas occasionnel selon le calendrier) = 13 cycles + 1 semaine hors cycle.
⏱ Règle de comptage des postes courts
Poste de 4h réelles → compté 6h pour le calcul des HS, par défaut. Ces valeurs (durée du poste, durée comptée, date d'application) sont configurables dans le sous-onglet 📅 Planning Annuel, et ne s'appliquent qu'aux postes correspondant exactement à la durée renseignée, à partir de la date choisie.
🗓 Comment lire le Planning Annuel
▸Chaque colonne = 1 semaine · Chaque ligne = 1 agent
▸Les semaines sont groupées par cycle (séparateur orange vertical)
▸La colonne Total H (droite) affiche le total annuel + le détail par cycle (C1, C2…)
▸Vert = dans les seuils
▸Orange = entre 134h et 151h → vérifier
▸Rouge ⚠ = dépassement — heures supplémentaires à déclarer
Exemple : Agent ANCEAUX — poste nuit 20h→08h = 12h/poste.
Si 3 postes/semaine sur 4 semaines → 12 × 3 × 4 = 144h → zone HS (134 → 151h) → 10h à majorer à 25%.
Si 4 postes/semaine → 12 × 4 × 4 = 192h → dépassement → 58h à majorer à 50%.

⚙️ Configuration
🔄 Synchronisation multi-appareils cliquer pour déplier ▾
Sites, statuts agents et commandes sont maintenant synchronisés via Supabase.
Pour activer la sync, vous devez créer une table krl_config dans votre Supabase.
📋 SQL à exécuter UNE SEULE FOIS dans Supabase → SQL Editor
CREATE TABLE IF NOT EXISTS krl_config (
  cle TEXT PRIMARY KEY,
  valeur TEXT,
  updated_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE krl_config ENABLE ROW LEVEL SECURITY;
CREATE POLICY "allow_all" ON krl_config FOR ALL USING (true) WITH CHECK (true);
💡 Chaque modification est envoyée automatiquement vers Supabase.
🔄 Un pull automatique toutes les 30 secondes met à jour les autres appareils.
🟢 Le point en haut à droite = état sync (vert = ok, orange = en attente, cliquer pour forcer).
